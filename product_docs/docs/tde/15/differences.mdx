## Limitations TDE imposes on standard Postgres functionality

If transparent data encryption is enabled, the `FILE_COPY` strategy cannot be used in `strategy` parameter with `create database`. See [PostgreSQL CREATE DATABASE documentation](https://www.postgresql.org/docs/15/sql-createdatabase.html) for more information.

## Postgres tools affected by TDE

You can enable and disable transparent data encryption for the following:

| Command/tool | Description | 

- [postgres](https://www.postgresql.org/docs/15/app-postgres.html) - The PGDATAKEYUNWRAPCMD environment variable 
- wal files - you can encrypt WAL files
  - [pg_waldump]() 
  - [pg_resetwal]()
- backups - if the backup is of encrypted database, you can decrypt the backup files
  - [pg_verifybackup]
  - [pg_rewind]

- pg_upgrade - upgrading the encrypted database cluster
