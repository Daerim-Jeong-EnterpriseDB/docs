EDB Pgpool-II can act as a middleware between your client application and BigAnimal cluster. To learn more, see [EDB Pgpool-II](../../../../pgpool/latest/).

For instructions on installing EDB Pgpool-II, see [Installing EDB Pgpool-II](../../../../pgpool/latest/01_installing_and_configuring_the_pgpool-II/).

## Configure EDB Pgpool-II

1. Open BigAnimal.

1. Go to **Clusters > Connect**.

1. Under **Connection Info**, find you cluster information.

1. Update the EDB Pgpool-II configuration file located at `/etc/pgpool2/pgpool.conf` according to the cluster connection information and your preferences:

   | Parameter | Value |
   | --------- | ----- |
   | `backend_hostname0` | Cluster hostname |
   | `backend_port0`   | Cluster port number |
   | `backend_weight0` | Load balance ratio |
   | `backend_data_directory0` | Data directory | 
   | `backend_flag0` | Set to `ALWAYS_PRIMARY`, `ALLOW_TO_FAILOVER`, or `DISALLOW_TO_FAILOVER`.  |
   | `allow_clear_text_frontend_auth` | Set to `on` to allow EDB Pgpool-II to copy the password in plain text from the frontend and use it for backend authentication. |

1. Restart EDB Pgpool-II. Enter the following commands:

   ```shell
   $ systemctl restart pgpool2
   $ systemctl status pgpool2
   ```

## Check the connection

1. Connect the database from the Pgpool server to the BigAnimal cluster with EDB Pgpool-II. Enter the following command:
   
   ```shell 
   $ psql -h localhost -p <port number> -U <username>
   ```

1. Enter the password. 

1. Connect the database from your personal machine to the BigAnimal cluster with EDB Pgpool-II. Enter the following command: 

   ```shell
   $ psql -h <hostname> -p <port number> -U <username>
   ```

1. Enter the password.
