After you have configured all of the Thales CipherTrust Manager keys as stated in the Configuring section, you will be able to use them in conjunction with EDB Postgres Advanced Server 15.2.

!!! Note
    It is important to note that this doc is intended for versions 15.2 of EDB Postgres Advanced Server as this version has the necessary addition of TDE.

Before you can implement Thales CipherTrust Manger with EDB Postgres Advanced Server 15.2 you must ensure that you have the following downloaded to your system:
- Python
- [pykmip](https://pypi.org/project/PyKMIP/#files)
- edb_tde_kmip_client.py

All of the `*.pem` files that you created above need to be copied to your system where your EDB Postgres Advanced Server 15.2 instance is installed.  For this guide, all of the `*.pem` files are in the `/tmp/` directory as well as the `edb_tde_kmip_client.py` file.

## Create pykmip.conf File

1. On your system where you have your EDB Postgres Advanced Server 15.2 instance, navigate to the directory where you have saved your *.pem files and the edb_tde_kmip_client.py client.

2. In that directory create a file called `pykmip.conf` and input the following:
- Host
- Port
- Username
- Password
- Keyfile
- Certfile
- Ca_certs

## Create a Key on Thales CipherTrust Manager

1. On your system where you have your EDB Postgres Advanced Server 15.2 instance, assume root user to create the key on the Thales CipherTrust Manager.

2. Type `python3` and then input the following, making adjustments per your system setup and directory paths:

```bash
>>> from kmip.pie import client
>>> from kmip import enums
>>> c = client.ProxyKmipClient(config_file='/tmp/pykmip.conf')
>>> c.open()
>>> key_id = c.create(enums.CryptographicAlgorithm.AES, 128, name='edbtestkey')
>>> c.activate(key_id)
>>> key_id
>>> 'key_output_shows_here'
>>> c.close()

```
3. Navigate back to Thales CipherTrust Manager and check that the key you created, in this case `edbtestkey` has been created.

## Verify Encryption and Decryption

To ensure that your key that you created will be able to encrypt and decrypt data, run the following two commands as the root user on your system where you have your EDB Postgres Advanced Server 15.2 instance.

1. `printf secret | python3 /tmp/edb_tde_kmip_client.py encrypt --out-file=test.bin --pykmip-config-file=/tmp/pykmip.conf  --key-uid='key_output_here’ --variant=thales`

2. `python3 /tmp/edb_tde_kmip_client.py decrypt --in-file=test.bin --pykmip-config-file=/tmp/pykmip.conf --key-uid='key_output_here' --variant=thales`

3. If this is successful it should produce the output of **secret**.

## Perform initdb for the Database

After you have completed the above steps you will be able to export the PGDATAKEYWRAPCMD and PGDATAKEYUNWRAPCMD to wrap and unwrap your encryption key and initialize your database.

1. Login to your EDB Postgres Advanced Server system.

2. Navigate to the `/bin` directory where your executables live, for example `/usr/lib/edb-as/15/bin` would be the executables directory for an Ubuntu system.

3. `export PGDATAKEYWRAPCMD='python3 /tmp/edb_tde_kmip_client.py encrypt --pykmip-config-file=/tmp/pykmip.conf --key-uid=key_ouput_here --out-file=%p --variant=thales’`

4. `export PGDATAKEYUNWRAPCMD='python3 /tmp/edb_tde_kmip_client.py decrypt  --pykmip-config-file=/tmp/pykmip.conf   --key-uid=key_output_here  --in-file=%p --variant=thales’`

5. Perform your initdb per your database requirements, for example: `./initdb -D dd12 -y`.
